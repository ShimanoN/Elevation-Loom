# 今後の展望

- リント / フォーマット
  - `npm run format` と `npm test` を実行して未解消の ESLint エラーを修正する。
  - CI（GitHub Actions）で自動実行させる。

- Husky / pre-commit
  - フックで不要にコミットが止まらないようルールとオーバーライドを整備する。

- E2E と CI
  - Playwright のブラウザキャッシュ/インストールを CI に最適化する。
  - フォールトを分離して実行時間を短縮する。

- カバレッジ / アーティファクト
  - `coverage/` をコミットに含めるか `.gitignore` で除外する方針を決める（通常は除外）。

- ドキュメント
  - `CONTRIBUTING.md` と開発セットアップ手順を追加する（ローカルでの Playwright セットアップ等）。

- リリース / 運用
  - リリース手順、バージョニング方針、既知の問題の追跡方法を整備する。

- 優先順位（短期）
  1. lint 整備とテストの安定化
  2. CI 改善（Playwright 実行・キャッシュ）
  3. ドキュメント整備
  4. 必要なリファクタリングとテスト追加
